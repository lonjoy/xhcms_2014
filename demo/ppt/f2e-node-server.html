• f2e-node-server 为前端打造的自由化开发服务器

    功能概述

    1. 基本模板引擎 & 文本文件的包含和布局 笔记
    2. less & coffee的动态解析，css、js类型文本动态压缩，
    3. 支持通过配置，批量输出以上动态资源结果到指定文件夹。
    4. GET、POST请求参数，文件上传等支持，
    5. 比较完整的远程数据代理交互支持，
    6. 浏览器客户端代码预览，PSD文件预览【部分浏览器】。

  • 安装环境 (安装nodejs&npm以及f2e-node-server)

    了解nodejs环境

    1. nodejs官网下载安装 (for Windows)
    2. 检查nodejs以及npm环境 (DOS操作)
    3. 安装f2e-node-server [npm install f2e-node-server]
    4. 如果你看到以下信息,则表示安装成功: 恭喜！

  • 基本配置 (简单配置项功能)

    配置根目录, 常用配置功能项等 nodeLib/config/conf.js

    1. 每一组配置分别对应一个独立的项目
    2. root:项目根目录, port:服务端口, maxConnections: 最大并发链接数
    3. handle:开启页面包含功能
    4. runJs: 支持模板功能 (依赖开启handle)
    5. less或者coffee的功能和handle并列

  • 完整配置 (扩展配置项功能)

    配置特殊页面,资源文件常用配置,服务器代理等扩展配置 nodeLib/config/conf.js

    1. 404, 欢迎页面, 文件夹列表展示页(需要开启runJs配置) 配置
    2. debug: 开启后将不对资源文件进行压缩
    3. fs_mod: 是否开启文件夹目录(需要配置folder)
    4. output: 将页面包含/less/coffee等解析结果,批量输出到指定文件夹 pluhin:build
    5. agent: 根据path路径分发代理请求 完整配置
        root :"D:\\WORK\\XhcmsProject\\Tomcat\\webapps\\xhcms\\xhcms_2014\\", //服务器索引的根目录，可配置为任意本地地址
    welcome: "",    //使用欢迎页面的文件名，为空时，表示不使用欢迎页面
    notFound: __dirname + "/../html/404.html",      //访问的资源不存在是，跳转的页面配置
    folder: __dirname + "/../html/folder.html",     //显示文件夹列表时候的配置页面
    handle: true,       //是否使用服务器模板引擎
    coffee: true,       //是否支持coffee-script动态解析
    less: true,         //是否支持less动态解析
    debug: true,        //是否对js以及css文件进行简单压缩，debug:true表示不压缩
    fs_mod: true,       //是否支持文件夹列表展示
    port: 80,           //服务器监听端口
    maxConnections: 1000,    //并发处理的最大连接数
    runJs : true,
    output: "c:\\output\\",  //静态化输出目录
    agent : {
        get:function(path){
            return /static/.test(path) ? {
                    reg : /static/,
                    host: 'localhost',
                    port: 2850,
                    path: function(url){
                        return '/index.html';
                    }
                } : false;
        }
    },
    expires : 0     //服务端缓存时间设置

  • 其他功能 (服务器功能扩展)

    你可以参考样例开发属于自己的插件

    1. /upload: 文件上传,支持配置文件上传目录以及返回数据格式配置
    2. /combine: 专门为支持KISSY模块动态资源合并而开发的plugin
    3. /psd：为客户端预览PSD文件做的简单支持
    4. /prettify：为客户端预览文本文件做的简单支持
    5. /agent：早期使用的最简单GET请求代理实现
    6. /favicon：favicon.ico的简单实现
    7. /build：项目输出插件
